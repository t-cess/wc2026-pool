# 📐 แผน: ระบบ Realtime Auto + แอดมินเต็มรูป

## 🎯 เป้าหมาย
1. **Realtime auto**: ผลบอล + คนยิง อัพเดตเองไม่ต้องมีคนกรอก (ESPN + Qwen + พจนานุกรม)
2. **แอดมินเต็มรูป**: แอดมินทำเองได้ครบในแอป (เผื่อ auto พลาด / อยากแก้)

---

## 🔑 จุดตัดสินใจสำคัญ: รวมโมเดลคะแนนของวง 1 ให้เป็น "auto-compute" (เหมือนวง 2)

**ตอนนี้:** วง1 = carry-driven (ตารางคะแนน = ยกมา + แชมป์, ผม push ยอดเอง) · วง2 = auto-compute (= ยกมา + แต้มรายคู่ + แชมป์, แอปคิดเอง)

**ปัญหา:** auto-grader เขียน "ผลรายคู่" → แต่วง1 ไม่เอารายคู่มาคิด → ตารางไม่ขยับ

**ทางแก้ (แนะนำ):** เปลี่ยนวง1 เป็น auto-compute เหมือนวง2
- **ยกมา (carry) = แช่แข็งยอดปัจจุบัน** (37/35/35/33/33/31) เป็นฐานตั้งต้น
- จากนี้ทุกคู่ที่ตรวจ (auto หรือแอดมิน) → แต้มรายคู่บวกเพิ่มเอง → ตารางขยับเอง
- "วันนี้" = แต้มจากคู่ที่จบวันนี้ · ลูกศร = เทียบเมื่อวาน (auto)
- **ไม่ double count** เพราะ carry = ยอด ณ ตอนนี้ คู่เก่าอยู่ในฐานแล้ว คู่ใหม่ค่อยบวก
- ⚠️ เป็นการแก้วงที่ใช้งานจริง → **เทส localhost ให้ยอดตรงก่อน deploy**

→ ผลลัพธ์: วง1 กับ วง2 ใช้สถาปัตยกรรมเดียวกัน · auto-grader + แอดมิน ป้อนที่เดียว ตารางคิดเอง

---

## Part A — ระบบ Realtime Auto (auto-grade.mjs ✅ ร่างแล้ว)
**ไหล:** `cron → ESPN API (สกอร์+คนยิง จริง) → จับคู่ทีม EN↔ไทย → เขียนผลลง Firestore → ตรวจคนยิง → แอปคิดแต้มสด`

**ตรวจคนยิง 3 ชั้น:**
1. พจนานุกรม `aliases.json` (Claude ตรวจแล้ว) → auto
2. Qwen เจาะจง ("X = คนยิงจริงไหม?") สำหรับชื่อใหม่ → auto
3. ฉายาที่ Qwen ไม่รู้ → แอดมินแก้ในแอป
+ **self-learning**: ชื่อใหม่ที่ตัดสินได้ → เติม aliases.json อัตโนมัติ (ยิ่งใช้ยิ่งไม่ต้องมีคน)

**Cadence:** cron ทุก 5 นาที — แต่ลงมือเฉพาะคู่ที่ ESPN บอกว่า "จบ" แล้ว (ไม่จบไม่แตะ)

**Realtime สกอร์สด (option):** ถ้าอยากเห็นสกอร์วิ่งระหว่างเกม → เพิ่มสถานะ "live" ในแอป + ESPN เขียนสกอร์สดทุก 5 นาที (เฟสหลัง)

**ต้องเคาะ:** cron รันเครื่องไหน (ต้องเปิดช่วงบอลเตะ + มี node + serviceAccount + claude-9arm)

## Part B — แอดมินเต็มรูป (วง1) — ยกมาจากวง2
- รายชื่อวง + คะแนนยกมา (ตั้ง/แก้)
- เพิ่ม/แก้/ลบคู่ · กรอกผล + **ติ๊กคนยิงรายคน** (เผื่อ auto พลาด)
- ตั้งแชมป์ · ล็อกทายแชมป์ · ป้ายชุดคืน→เช้า
- จัดการสมาชิก (ดู/ปลดการจับคู่)

## 🛠️ ลำดับสร้าง (เฟส)
1. **เปลี่ยนวง1 → auto-compute** (แก้ renderBoard/computeBoard ให้เหมือนวง2) → เทส localhost ยอดตรง → deploy
2. **ใส่แอดมินเต็มรูปในวง1** (port จากวง2) → เทส → deploy
